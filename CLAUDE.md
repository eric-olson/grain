## Build & Run

```bash
cargo build --release        # Build optimized binary → target/release/grain
cargo run --release          # Build and run
cargo build                  # Debug build (faster compile, slower runtime)
cargo fmt                    # Format all code
cargo clippy                 # Lint — treat warnings as errors
```

Run `cargo fmt` and `cargo clippy` before committing. No test suite yet — validation is done manually using files in `testdata/` (generated by `testdata/generate.py`).

## Architecture

Rust GUI application ("grain") for visualizing binary files, built for satellite radio data reverse-engineering. Uses **eframe/egui** (immediate-mode GUI) with **memmap2** for memory-mapped file access.

### Module Overview

- **`app.rs`** — Core state machine implementing `eframe::App`. Owns top-level state (file, stride, zoom, display mode, selection) and delegates UI rendering to submodules in `ui/`.
- **`ui/`** — UI panels extracted from `app.rs`. Each module renders one panel:
  - `menu_bar` — File open, display mode, panel toggles
  - `viewport` — Central pixel grid display via `PixelGridViewer`
  - `inspector` — Data inspector showing values at cursor in multiple numeric types
  - `search_panel` — Hex pattern search with variation generation
  - `stride_dialog` — Stride auto-detection UI
  - `status_bar` — File info, offset, cursor position
  - `processor_panel` — Pipeline processor add/remove/reorder UI
- **`viewer.rs`** — `PixelGridViewer` renders binary data as a 2D pixel grid. Two display modes: Byte (grayscale) and Bit (B/W per bit). Builds `ColorImage` on CPU, uploads as egui texture, caches until invalidated.
- **`pipeline/`** — Data processing pipeline that transforms bytes between file read and display:
  - `processor.rs` — `Processor` trait: `process()`, `ratio()`, `save_state()`/`restore_state()`, `show_config()`. Stateful processors support checkpointing.
  - `processors/` — Concrete implementations: `TakeSkip` (stride extraction), `BitShift`, `NrzM` (NRZ-M decoding, stateful)
  - `node.rs` — `PipelineNode` wraps a processor with parent/child links and checkpoint storage
  - `cache.rs` — `OutputCache` for chunk-aligned caching of processed output
  - `offset_map.rs` — `OffsetTranslator` maps between input/output byte offsets through ratio chains
  - `checkpoint.rs` — Periodic state snapshots for stateful processors enabling fast seeking
- **`file_handler.rs`** — `MappedFile` wrapper around `memmap2::Mmap`. Provides `get_range()` for bounded slice access.
- **`stride_detect.rs`** — Stride detection via byte/bit-match periodicity with z-score statistics and harmonic filtering.
- **`sync_search.rs`** — Hex pattern search with automatic variation generation (bit-inverted, bit-reversed, byte-swapped).
- **`types.rs`** — Shared data types: `CursorInfo`, `Selection`, `InspectType`.

### Key Patterns

- **Threading**: Heavy operations (search, stride detection) run on `std::thread::spawn` with `mpsc::channel` for results. Main thread polls with `try_recv()` each frame.
- **Pipeline processing**: `Pipeline::get_range()` chains processors, using `OffsetTranslator` to compute input ranges from output ranges. Stateful processors (NRZ-M) warm up from file start or nearest checkpoint.
- **Viewport rendering**: Only visible data is rendered. Texture is rebuilt only when offset/stride/zoom/highlights change.
- **Bit-addressability**: Stride and display can operate at bit granularity for non-byte-aligned satellite protocols.
- **Data flow**: `MappedFile` → `Pipeline::get_range()` (if active) → `PixelGridViewer` renders visible window.

### Adding a New Processor

1. Create `src/pipeline/processors/my_proc.rs` implementing the `Processor` trait
2. Register it in `src/pipeline/processors/mod.rs`
3. Add it to the processor picker in `src/ui/processor_panel.rs`
